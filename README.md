# investment-alarm-checker

한국 주식 시장의 **투자경고종목** 다음 거래일 상태를 자동으로 분석하는 CLI 도구.
Naver Finance 공개 API만 사용 (외부 의존성 없음).

장 종료 이후 (저녁 9시 등) 실행하여 다음 거래일에 해당 종목이:
- 이미 해제 상태인지
- 곧 해제 예정인지
- 판단일에 도달하여 해제 조건이 적용되는지 (종가 임계치 계산)
- 아직 판단일 전인지 (대기 중)

를 한 줄 명령으로 확인할 수 있습니다.

## 요구사항

- Python 3.8+
- 외부 패키지 불필요 (`urllib`, `json`, `re` 등 표준 라이브러리만 사용)

## 사용법

### CLI

```bash
python3 check_warning.py "<종목명>"
```

### 모듈 임포트

```python
from check_warning import check_warning_stock

print(check_warning_stock("드림시큐리티"))

# 특정 기준일로 백테스트
from datetime import date
print(check_warning_stock("이오테크닉스", today=date(2026, 5, 14)))
```

## 동작 원리

### 데이터 소스 (Naver Finance, 모두 공개 엔드포인트)

| 용도 | API |
|------|-----|
| 종목명 → 종목코드 | `ac.stock.naver.com/ac?q={name}&target=stock` |
| 공시 목록 | `m.stock.naver.com/api/stock/{code}/disclosure` |
| 공시 상세 (HTML) | `m.stock.naver.com/api/stock/{code}/disclosure/{id}` |
| 일별 시세 OHLCV | `fchart.stock.naver.com/siseJson.naver` |

### 분류 로직

공시 제목으로 분류:
- `지정해제` 포함 → **release** (해제 류)
- `지정예고` 포함 → 무시
- `지정` 포함 (위 둘 제외) → **designation** (지정 공시)

최신 공시가 release면 본문에서 해제일 추출, designation이면 본문에서 최초 판단일·해제요건 %·매매거래정지 여부 추출.

### 해제 조건 (3가지 중 하나라도 미충족 시 해제)

판단일 T에 대해:
- **cond1**: T종가 < T-5종가 × (1 + pct1)
- **cond2**: T종가 < T-15종가 × (1 + pct2)
- **cond3**: T종가 < 최근 15거래일 종가 최고가

`binding = min(cond1, cond2, cond3)` 가 실질 해제 임계가.

`pct1`/`pct2`는 종목별로 다름 (45%/75% 또는 60%/100% 등). 공시 본문에서 정규식 파싱.

### 매매거래정지 (해제 조건과 별개)

판단일 T-2 종가 × 1.4 가 상한가 (오늘 종가 × 1.3, 호가단위 절사) 이내면 도달 가능.
"투자위험 해제 → 재지정" 케이스는 본문에 `별도의 매매거래정지` 명시 → 자동 스킵.

## 캐싱

종목명 → 종목코드 매핑은 스크립트와 같은 폴더의 `stock_codes.json`에 자동 캐싱.

## 면책

- Naver Finance 공개 API에 의존하므로 API 변경 시 동작 안 할 수 있음.
- 공시 본문 정규식 파싱은 KRX 표준 양식 기준. 양식 변경 시 파싱 실패 가능.
- 본 도구는 정보 제공용. 실제 매매 결정의 책임은 사용자에게 있음.
